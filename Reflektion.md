Vi har i denne opgave arbejdet med flere forskellige CSS-principper og prøvet at implementere dem i praksis i vores løsning.

## Reflektion

**Det der fungerede godt:**

- Vi fik en bedre forståelse for grid, subgrid og hvordan layout hænger sammen
- Vi arbejdede med moderne CSS (fx scroll animationer og variabler), som gjorde siden mere dynamisk
- God øvelse i at bygge komponent-baseret (Astro) og genbruge kode
- Vi blev bedre til at tænke responsivt fra start

**Det der var udfordrende / mindre godt:**

- Subgrid og mere avancerede layouts var svære at få til at opføre sig som forventet
- Safari / browser compatibility gav nogle problemer
- Nogle løsninger blev lidt "hacky" fordi vi skulle fixe ting hurtigt
- Det var til tider svært at holde struktur og overblik i CSS’en

**Hvad jeg tager med videre:**

- Jeg er blevet bedre til at debugge layout issues
- Jeg vil fremover være mere struktureret i min CSS fra start
- Jeg vil tænke endnu mere i komponenter og genbrug

Alt i alt en god opgave hvor jeg har lært meget, men også fået øjnene op for hvor vigtigt struktur og planlægning er i frontend udvikling.
